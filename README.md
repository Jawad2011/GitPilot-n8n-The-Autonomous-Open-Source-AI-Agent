## Its features:

#### 1) It constantly monitors the issues on a GitHub repository 24/7.
#### 2) It instantly notifies the developers about the issues.
#### 3) It can create new issues based on its comments and information.
#### 4) It requires zero human intervention.
#### 5) It generates a summary that helps developers better understand the issues.


## How it is built:

## Why isn't there any demoable link:

### n8n is a locally run AI Automation tool. It saves its credentials directly on the servers that you are hosting on. If I want to make it demoable, I will expose my credentials, which carries a significant risk. 

## How to run it yourself:

### First, copy the source code and paste it directly inside the n8n canvas. Then, insert your keys, credentials, and links of your own that aren't in the source code because it is exclusive to you. Then, insert System messages and descriptions, which are very important for the AI Agent to function properly. I made it a lot easier for you. Insert the system prompt in the exact named node of the file. Then, run it.
## How it works:

### It first tracks the issue 24/7 using the GitHub API, which sends a webhook to my n8n workflow. Then, it parses the necessary data, and the AI Agent processes it. Then, the Agent sends it to the developer. To keep the reporter of the issue satisfied, it also creates a comment notifying that the issue has been sent to the developer.
